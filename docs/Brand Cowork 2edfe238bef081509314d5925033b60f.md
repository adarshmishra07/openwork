## The Problem (In Simple Terms)

Imagine you run an online clothing store. Every day, you need to:

> Write product descriptions for 50 new itemsCreate social media posts that "sound like your brand"Check what your competitors are pricing similar items atUpdate your website banners for a saleReply to customer emails in your brand's voiceAnalyze which products are selling well
> 

**Today, you either:**

1. Do it all yourself (exhausting, slow)
2. Hire people (expensive, hard to train on "your voice")
3. Use ChatGPT (generic, doesn't know your brand, can't actually DO things)

## The Opportunity

**What if you had an AI assistant that:**

```
âœ… Learned your brand voice after onboarding
âœ… Could actually log into Shopify and make changes
âœ… Remembered what worked last time
âœ… Could research competitors by browsing their sites
```

# â“ FAQs

**Q: Why fork OpenWork instead of building from scratch?**

> OpenWork already solved the hard problems: Tauri shell, agent lifecycle, file management. We can focus on commerce-specific features.
> 

**Q: Why a desktop app instead of web?**

> Privacy (data stays local), performance (no latency), and system access (can run CLI tools, access files).
> 

**Q: How does brand memory work?**

> SQLite database stores brand context (voice, style, rules). Every agent prompt includes this context. Agent can also write back learned patterns.
> 

**Q: What if the agent makes a mistake?**

> Permission system catches dangerous actions. All changes are logged. Users can undo via Shopify admin.
> 

**Q: Can this work with non-Shopify stores?**

> MVP is Shopify-only. Future: WooCommerce, BigCommerce, custom stores via API.
>
